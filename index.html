Work

Check out my work below! Hopefully, it speaks for itself.
Click the images to head on over to my github, or to the pages themselves

RecipeShare: Back end engineer for ﬁrst and second feature canvas release for RecipeShare; a recipe app created with the user in mind.   RecipeShare used ReactNative and NodeJS (along with express) to create an app that would allow a user to create a local copy of popular recipes, edit them, and have their changes shared with the public and voted on so that the best possible version of a given recipe could make the way to public eye.  As one of three back-end developers, I was responsible for migrations and initial seed data, endpoint creation, deploying a postGRESQL database using Heroku, creating authentication middlewear using JSON Web Tokens, Db manipulation using knex (3rd party SQL query library), communicating changes with front end, and acting as a liaison between front and back end engineers, as well as writing tests using Jest and SuperTest, using POSTMAN and pgAdmin to test a local development environment.

MacroCalculator: "Build Week" project for Lambda School. Back end engineer on a team of two engineers for a week-long code sprint challenge to create a website that will take in a user's biometric information and produce a Macronutrient Proﬁle specﬁc to the user. This information would then be used to generate a weight loss plan. My team had one week to create a back-end server using Java/Spring with a full authentication suite (using web tokens and local storage, that could house a user's proﬁle including all their biometrics, that had full CRUD functionality.  Responsible for authentication, endpoints, and seed data. tested using POSTMAN and pgAdmin. seed data written in raw SQL 

ReceiptTracker: "Build Week" project for Lambda School. Front End engineer on a team of two engineers for a week-long code sprint to create a website that will allow a user to enter Receipts Images and Information  in order to track trends and spot changes over time.  All receipts were house in a custom built API designed by other Lambda School students. The front end used ReactJS with redux as a state management system to pull in user-related receipt data from two APIs (image data was housed and pulled from third party API called Cloudinary) and populate a receipt book feature similar to a user homepage.  Once populated, the user could ﬁlter through receipts by category, company, amounts spent or a speciﬁed time frame.  They could also, add, update, or delete receipts on the ﬂy.

Educational work on responsive design using Flexbox.

Proof of concept on using React.js with functional components and hooks, the API fetch call was made with the axios library. My Nasa photo of the day site was deployed using ZEIT!
